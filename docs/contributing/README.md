# Contributing to AcousticForge

We're excited that you're interested in contributing to AcousticForge! This document provides guidelines for contributing to our open source audio calibration platform.

## How to Contribute

### Reporting Issues

If you find a bug or have a feature request, please search the issues on GitHub to see if it has already been reported. If not, create a new issue with a clear title and description.

### Code Contributions

1. Fork the repository
2. Create a new branch for your feature or bug fix
3. Make your changes in the new branch
4. Write tests for your changes
5. Submit a pull request

### Code Style

We follow standard JavaScript/TypeScript coding conventions. Please ensure your code follows the existing style to maintain consistency.

## Development Setup

1. Clone the repository
2. Install dependencies: `npm install`
3. Run tests: `npm test`
4. Start development server: `npm run dev`

## Testing

We use a combination of unit tests and integration tests. Please ensure all tests pass before submitting a pull request.

## Code of Conduct

Please note that this project is released with a Contributor Code of Conduct. By participating in this project you agree to abide by its terms.

## Questions or Need Help?

If you have questions about the project or need help getting started, feel free to reach out via GitHub issues.

Thank you for your interest in improving AcousticForge!
